# Security Policy

This repository is intentionally built as a portfolio-grade engineering project.
Please do not open public issues for secrets, vulnerabilities, or private
infrastructure details.

## Supported branches

The `main` branch receives security fixes.

## Reporting

Email the maintainer with a short reproduction, affected component, expected
impact, and suggested remediation.

## Baseline controls

- Dependency scanning in CI.
- Static analysis where the language ecosystem supports it.
- Docker image hardening notes.
- No committed credentials or production endpoints.
